# 모두마켓 월간 로그 — 도구 선택 보고서

## 1. 데이터 개요
- 파일: web_logs.csv
- 크기: 101.6 MB, 500000 행
- 주요 컬럼: ts / user_id / page / device / status_code / response_ms / bytes_sent

## 2. 측정 결과
| 방식 | 소요 시간 | 메모리 피크 |
|---|---|---|
| pandas + dtype | 2.32초 | 2.0MB |
| pandas chunked | 2.31초 | 32.5MB |
| Polars lazy    | 0.31초 | 2.2MB |

## 3. 분석 결과 요약
- 페이지별 평균 응답 시간 최고: cart (160.27ms)
- 디바이스 점유율: mobile 69.9%, desktop 25.1%, tablet 4.9%
- 에러 1위 페이지: home (42898건)

## 4. 도구 선택 정당화 (한 단락)
이 분석에는 **polars (lazy)** 을 선택했습니다. 이유는
(1) 소요 시간이 가장 짧았고 (0.31초, pandas 대비 약 7배 빠름), 메모리 사용량도 가장 낮았습니다 (2.2MB). 또한 (2) 팀 환경상 앞으로 데이터 크기가 계속 늘어날 가능성이 높아, 처리 속도와 메모리 효율이 중요합니다. (3) lazy evaluation 방식이라 불필요한 연산을 최적화해서 실행하기 때문에, 대용량 데이터에서 특히 유리합니다.
