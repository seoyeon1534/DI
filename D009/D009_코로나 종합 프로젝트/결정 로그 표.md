| 단계 | 액션 | 근거 | 결과 |
|---|---|---|---|
| 1. 중복 제거 | 완전 중복 행 drop | 동일 키·동일 값 중복은 시스템 입력 오류로 간주 | 13건 제거 |
| 2. 문자열 정제 | 기준시 표기 통일 (`'18'`, `'18tl'` → `'18시'`) | 서로 다른 표기가 실질적으로 같은 범주를 가리켜 집계 왜곡 위험 | 기준시 3개 범주(`18시`/`10시`/`17시`)로 정리 |
| 3. 날짜 파싱 | 기준일을 datetime으로 변환 (공백은 하이픈으로 교정 후 파싱) | 연도·월별 집계가 분석 목적이므로 datetime 필요 | 공백 교정으로 1건 복구, 일자 정보 자체가 없는 14건은 파싱 실패(NaT)로 제거 |
| 4. 이상치 처리 | 신규확진자 수 이상치(IQR)를 삭제하지 않고 `is_outlier` 플래그로 표시 | 집단감염 등 실제 급증일 가능성이 높아 제거 시 중요한 신호 손실 우려 | 이상치 162건 플래그 처리 |
| 5. 결측치 처리 | 사망자 수 결측 → forward fill로 복구, 자가격리인원 결측 → 유지 + 측정여부 플래그 추가 | 사망자 수는 누적값이라 전날 값 이어쓰기가 타당하며, 자가격리인원 결측은 정책 변화로 인한 미집계로 추정되어 0으로 채우면 왜곡 위험 | 사망자 수 310건 복구, 자가격리인원 534건은 결측 유지 + 플래그 추가 |
