# 서울 강동구 코로나 현황 데이터 정제·집계 보고서

## 1. 데이터 개요
- 출처: 공공데이터 포털의 (`서울특별시_강동구_코로나현황_구_단위__20230901__1_.csv`)
- 한 행이 의미하는 관측 단위: 강동구의 특정 날짜(기준일)·특정 발표 시각(기준시) 기준 코로나 현황 집계 1건
- 기간: 2020-02-07 ~ 2023-08-27 (2023년은 8월까지의 부분 데이터)
- 원본 행 수/ 정제 후 행 수: (1,281행) → (1,254행)
- 주요 컬럼의 의미: 
  1. 기준일 = 집계 기준 날짜　
  2. 기준시 = 집계 발표 시각(범주형)　
  3. 신규확진자 수 = 해당일 신규 확진자　
  4. 누적확진자 수 = 누적 확진자　
  5. 신규완치자 수 = 해당일 신규 완치자　
  6. 누적완치자 수 = 누적 완치자　
  7. 사망자 수 = (확인 결과 신규가 아닌 **누적** 사망자)　
  8. 자가격리인원 = 해당일 자가격리 인원
- 분석에서 특히 중요하게 볼 컬럼: 연도별·시기별 확산 추이 분석이 목표이므로 신규확진자 수, 사망자 수(누적사망자수), 신규완치자 수를 집중적으로 봄

## 2. 데이터 품질 진단
- 결측치가 있는 컬럼: 자가격리인원(41.69%), 사망자 수(24.20%), 누적완치자 수(2.11%), 신규확진자 수(0.16%)
- 중복 행 개수: 완전 중복 13건
- 수치형 컬럼의 이상치: 신규확진자 수(IQR 13.21%), 신규완치자 수(7.57%), 자가격리인원(2.95%) — 팬데믹 특성상 실제 급증일 가능성이 높아 무조건 제거는 지양
- 날짜·숫자로 보이지만 문자열로 들어온 컬럼: 기준일(maybe_datetime=True → datetime 필요), 누적확진자 수·누적완치자 수(순수 숫자인데 object로 읽힘)
- 같은 의미인데 다르게 적힌 값: 기준시 컬럼의 `'18'`, `'18tl'`이 `'18시'`와 같은 의미 (베이커리 데이터의 '케이크'/'케익' 사례와 동일한 유형의 표기 혼재)

## 3. 정제 과정과 판단 근거

> 사망자 수 결측은 전체의 약 24%였는데, 값을 직접 확인해보니 신규가 아닌 누적값이었다.
> 그래서 0으로 채우지 않고 전날 값을 이어쓰는 방식(forward fill)으로 채웠다.

1. **중복 행 처리**: 완전 중복 13건 제거 → 모든 이후 통계의 기준점이 흔들리지 않도록 가장 먼저 처리
2. **문자열 공백 제거**: 기준일 문자열의 공백(`"2021-02 27"`)을 하이픈으로 교정 → 날짜 파싱이 가능해짐
3. **표기 통일**: 기준시의 `'18'`, `'18tl'`을 `'18시'`로 통일 → 실질적으로 동일한 범주를 하나로 정리
4. **날짜 형식 변환**: 기준일을 str → datetime으로 변환, 그중 공백 교정으로도 복구 불가능한 14건(일자 정보 자체가 없는 값)은 제거
5. **이상치 처리**: 신규확진자 수 이상치(13.21%)는 삭제 대신 `is_outlier` 플래그로 표시 → 집단감염 등 실제 현상일 가능성이 높아 제거하면 중요한 신호를 잃을 위험이 있기 때문
6. **결측치 처리**: 사망자 수(24.2%)는 누적값임을 확인했으므로 forward fill로 채움. 자가격리인원(41.69%)은 결측이 2022~2023년에 집중되어 있어 "0명"이 아닌 "정책 변화로 미집계"로 판단, 0 대체 대신 결측을 유지하고 측정 여부 플래그만 추가함
7. **정제 후 저장 방식 선택**: 분석용 전체 데이터(`covid_clean`, 1,254행×12열)는 Parquet으로 저장 (컬럼 타입이 유지되고, 이후 컬럼 선택적 로딩이 가능해 재사용에 유리). 반면 4개 행짜리 `yearly_kpi` 같은 초소형 요약표는 Parquet의 메타데이터 오버헤드가 오히려 CSV보다 더 커서, 이런 경우엔 CSV가 더 적합함 (5번 항목에서 실측 비교)

## 5. 한계와 후속 질문

- **이 데이터만으로 단정할 수 없는 것**: 신규완치자 수가 2022년부터 0인 이유, 자가격리인원 결측이 정책 변화 때문인지는 이 CSV만으로는 추정일 뿐이며 공식 근거 확인이 필요하다. 2023년도 8월까지의 데이터라 연간 비교에 온전히 쓸 수 없다.
- **추가 데이터가 있다면 확인하고 싶은 것**: 서울시 전체 또는 다른 자치구 데이터와 비교해 강동구의 확산·치명률 패턴이 특이한지, 백신 접종률 데이터와 결합해 치명률 하락 시점이 실제로 접종률 상승과 맞물리는지 확인하고 싶다.
- **가장 조심해서 해석해야 하는 부분**: 사망자 수를 forward fill로 채운 값은 "전날과 같다"는 가정에 불과해 실제 사망 시점보다 다소 늦게 반영될 수 있다. 또한 신규확진자 수의 이상치(13.21%)는 삭제하지 않았기 때문에, 평균이나 합계를 볼 때 급증일의 영향이 크게 반영되어 있다는 점을 감안해야 한다.
