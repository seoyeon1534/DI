# 빵셀러 데이터 정제·집계 보고서

## 1. 데이터 개요
- 출처: (가상) 빵셀러 베이커리 체인 운영 로그
- 기간: 2025-04-01 ~ 2025-06-30 (3개월)
- 원본 행 수 / 정제 후 행 수: (2164)행 → (1842)행

## 2. 발견한 품질 문제
- 결측: revenue 컬럼에서 2.77%
- 중복: 완전 중복 행 312 건
- 이상치: quantity 컬럼에서 14.56, 
          revenue 컬럼에서 17.3%  (결측이 있어 추후에 다시 확인해야 할 필요 있음)
- 표기 혼재: date_str 컬럼의 dtype이 str이므로 datetime으로 전환 필요

## 3. 처리 결정과 근거 (5줄로)
1. 중복 행 (312)건 → 매출 합계 분석이 목표이므로, 완전 중복은 제거함
2. store_id 공백 → 앞뒤 공백은 제거함
3. item '케익' → '케이크' → 올바른 표준어 표기인 '케이크'로 바꿈
4. revenue 음수 → 환불로 판단 후, 별도 보관 (1)건 
5. revenue 결측 (60)건 → 재계산 (이유: quantity와 unit_price는 오염되지 않았으므로, revenue가 NaN인 60건도 사실 계산 가능한 값이기 떄문에 재 계산 후 그래도 남은 결측이 있다면 그때 제거 함)

## 4. 주요 KPI 결과 (2줄)
- (월·매장 관점) 가장 매출이 큰 매장과 그 월: S06 매장, 2025년 4월
- (상품 관점) 매출 1위 상품: total_revenue 기준으로 쿠키가 26,812,000원으로 1위
